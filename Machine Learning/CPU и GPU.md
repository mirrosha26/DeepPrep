# CPU и GPU (по сути)

## CPU

Мало ядер (2–64). Делает всё по очереди: одна сложная операция за другой. Универсален, но параллелизма почти нет. Подходит там, где шаги зависят друг от друга и порядок важен.

## GPU

Очень много слабых ядер (сотни–тысячи). Одно ядро медленнее CPU, но все вместе делают массу простых операций одновременно. Память GDDR под большие объёмы данных. Заточен под однотипные параллельные задачи: графика, матрицы, нейросети.

## В чём разница

- **Ядер:** CPU единицы-десятки, GPU сотни-тысячи.
- **Частота:** CPU выше, GPU ниже.
- **Модель работы:** CPU последовательно, GPU параллельно.
- **Энергия:** GPU обычно прожорливее.
- **Ниша:** CPU общие задачи и сложная логика, GPU графика, видео, ML.

## Зачем GPU в ML

Нейросети и тяжёлые модели это в основном много одинаковых операций над матрицами. Их удобно раздать по тысячам ядер GPU и считать параллельно. Плюс быстрый доступ к большим данным. Фреймворки (PyTorch, TensorFlow и т.д.) умеют считать на GPU через CUDA/cuDNN. Поэтому то, что на CPU тянется днями, на GPU часто укладывается в часы или минуты.
